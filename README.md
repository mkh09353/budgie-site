# budgie.to

Marketing site for [Budgie](https://github.com/mkh09353/budgie) — a tiny,
fully-offline, open-source dictation app for macOS.

Pure static: one `index.html` (inline CSS/JS), no build step, no dependencies.
The download buttons resolve the latest DMG from the GitHub Releases API at
runtime, so shipping a new app release updates the site automatically.

## Develop

```sh
python3 -m http.server 8077
# open http://localhost:8077
```

## Deploy

**Cloudflare Pages**: create a Pages project from this repo — framework preset
*None*, build command *(empty)*, output directory `/`.

**Vercel**: `vercel --prod` from the repo root, or import the repo in the
dashboard — framework preset *Other*, no build command, output directory `/`.

Then point **budgie.to** at it (add the domain in the Pages/Vercel project
settings and follow the DNS instructions they print — usually a CNAME on the
apex via CNAME-flattening, plus `www`).

## Assets

- `assets/` — favicons + app icon PNGs (generated by `tools/make-icon.swift`
  in the app repo) and `og.png`, the social card.
- `og.html` — source for the social card; screenshot it at 1200×630 to
  regenerate `assets/og.png`.
- The inline bird SVG uses the exact shape geometry from the app's
  `MenuBarIcon.swift`, so the site and the app are always the same bird.
